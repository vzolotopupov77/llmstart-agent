## Рамка от эксперта (постановка)

По `project-draft.md` агент должен проверять способности:

1. Понять запрос и уточнить потребность
2. Определить сегмент **B2B / B2C**
3. Подобрать продукт из B2C-каталога (combo, intensive, fullstack, agents, deep-agents, consultation)
4. Ответить через RAG по материалам `data/b2c/`
5. Предложить покупку (мок-ссылка), подтвердить оплату, зафиксировать лид

Ожидаемые типы запросов (гипотеза до анализа): формат/расписание, выбор курса, сравнение продуктов, цена/возврат, «подойдёт ли мне», корпоративное обучение vs физлицо.

## Таксономия групп запросов

Группы упорядочены по **охвату в корпусе** (сколько чатов из 5 затрагивают тему).

### G1. Формат обучения и расписание — **4/5 чатов** 🔴 ядро

**Что проверяем:** умеет ли агент из RAG ответить про live/запись, длительность, дни/время, доступ к материалам.

| Внутренняя категория | Примеры из данных | Чаты |
|---------------------|-------------------|------|
| **G1.1** Запись vs live / свой темп | «в своё удобное время?», «синхрон важен, видео не посмотрю» | 0014, 0020, 0110 |
| **G1.2** Конкретное расписание (дни, часы, длительность) | «по каким дням и во сколько», «до 2х часов», «вт/чт 16–18» | 0014, 0020, 0110 |
| **G1.3** Расписание ещё не определено (TBD) | «время пока не определили», «вечернее либо выходные» | 0110 |
| **G1.4** Timezone / travel | «в эту субботу я в Сан-Франциско», «10 утра по мск» без перевода | 0020 |
| **G1.5** Состав комбо / цепочка курсов | Fullstack → Agents → Deep Agents + бонус intensive | 0014 |
| **G1.6** «Не нашёл на сайте» | Пробел UX/RAG — ответ должен закрывать FAQ | 0014 |

**Частота в корпусе:** доминирующая группа; без неё не закрывается pre-purchase в большинстве чатов.
