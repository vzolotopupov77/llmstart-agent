## 5. Формат записи (ChatML)

```jsonc
{
  "id": "b2c-rag-001",
  "input": "Добрый день. Заинтересовал комбо курс. Какой формат? Можно в своё время?",
  "expected_output": {
    "segment": "b2c",
    "product_codes": ["ai-agents-combo"],
    "answer_key_points": [
      "онлайн и/или записи в своём темпе",
      "состав комбо: fullstack-aidd, agents, deep-agents + бонус intensive",
      "поддержка в чате, доступ к материалам"
    ],
    "should_clarify": false,
    "tools": []
  },
  "metadata": {
    "dataset_type": "b2c-rag",
    "group": "G1",
    "category": "G1.1",
    "source": "extraction",
    "source_chat": "CHAT_0014",
    "turn_mode": "single",
    "difficulty": "medium"
  }
}
```

**Multi-turn** (~20%): `input` — массив `{role: "user"|"assistant", content: "..."}` (2–4 сообщения); эталон — ответ на **последнюю** реплику user с учётом контекста. `metadata.turn_mode`: `"multi"`.

```jsonc
{
  "id": "b2c-objection-m01",
  "input": [
    {"role": "user", "content": "В каком формате занятия?"},
    {"role": "assistant", "content": "…запись 2025, комбо 2026 — онлайн-эфиры…"},
    {"role": "user", "content": "Покажите отрывок урока — не понимаю, за что платить такие деньги"}
  ],
  "expected_output": {
    "segment": "b2c",
    "answer_key_points": [
      "публичного демо / отрывков уроков нет",
      "описание программы из каталога",
      "не обещать ссылку на демо"
    ],
    "should_clarify": true,
    "acceptable_clarifications": ["какой курс интересует"]
  },
  "metadata": {
    "dataset_type": "b2c-objection",
    "group": "G4",
    "category": "G4.1",
    "source": "extraction",
    "source_chat": "CHAT_0110",
    "turn_mode": "multi"
  }
}
```

Для `b2c-tools` / tool-flow в `expected_output.tools`:

```jsonc
"tools": [
  {"name": "search_knowledge_base", "args": {"query": "...", "audience": "b2c"}},
  {"name": "create_payment_link", "args": {"product_code": "agents"}}
]
```

## 6. Метрики (проектируем вместе с эталоном)

| Тип | Поля эталона | Метрика | Как считаем |
|-----|--------------|---------|-------------|
| `b2c-segment` / `b2b-segment` | `segment` | **Segment Accuracy** | exact match `b2c` / `b2b` |
| `b2c-product` | `product_codes` | **Product Match** | set overlap / primary code exact |
| `b2c-rag` / `b2b-rag` | `answer_key_points` | **Answer Correctness** | LLM-judge (0/0.5/1.0 по покрытию пунктов) |
| `b2c-objection` | `should_clarify`, `acceptable_clarifications` | **Clarification Score** | 0 / 0.5 / 1.0 (skill metrics.md) |
| `b2c-tools` | `tools[]` | **Tool Call Accuracy** | name + args match (Evaluation) |
| ~~RAG retrieval~~ | ~~`chunk_ids`~~ | ~~Hit Rate@k~~ | **отложено до v2** (chunking не фиксируем в v1) |

**Не метрики (только metadata):** `difficulty`, `persona`, `source_chat`, `turn_mode` — аналитика, не score.

**v1 eval:** Answer Correctness, Segment/Product Match, Clarification Score, Tool Call Accuracy — **без** retrieval-метрик.

**Reference-free (v2):** длинные nurture-траектории G8 — рубрика + judge по всей цепочке.

### Политика G4: демо-контент ✅

**Решение (2026-06-09):** публичных фрагментов уроков **нет** — агент отвечает честно, без выдуманных ссылок.

**Эталон `answer_key_points` для G4.1 / G4.4 (запрос демо / senior-skeptic):**

1. Явно: публичного демо / отрывков уроков **нет**
2. Можно дать из RAG: описание программы, outcomes, сравнение курсов (`courses-overview`, program md)
3. При необходимости: политика возврата (если есть в KB) — **не как замена** демо, а доп. факт
4. **Не делать:** обещать «пришлю ссылку», «подумаю о демо», выдуманные preview URL
5. После отказа по демо — мягко уточнить интересующий курс (G2.4), **не** давить на «цель и опыт» после явного отказа (урок CHAT_0110)

**Метрика:** Rejection Accuracy на фразу отказа + Answer Correctness по key_points (нет галлюцинации демо).

## 8. План валидации

| Проверка | Кто/что | Когда |
|----------|---------|-------|
| Структурная | скрипт: поля, типы, уникальные id | после каждой партии |
| Покрытие | матрица group×type — нет пустых P0/P1 | перед v1 freeze |
| Семантическая | key_points сверяются с `data/b2c/` или `data/b2b/` | выборочно |
| Validation Sample | **человек** — 10% записей, естественность формулировок | ▶ стоп перед freeze |
| Strong model pass | прогон GPT-4 class по 20% — плохой score → проблема в датасете | перед freeze |
