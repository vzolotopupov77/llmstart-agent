# Onboarding: Continue Project

> **Режим:** проект уже на методологии, нужно открыть следующую задачу.
> **Скопируй этот промпт и отправь агенту как первое сообщение.**

---

## Промпт для агента

```
Продолжаем работу над проектом по методологии AI-Driven Development.

Методология: .methodology/METHODOLOGY.md
Правила процесса: .methodology/rules/20-workflow.mdc

Сделай следующее:

1. Прочитай docs/roadmap.md — определи текущий этап.

2. Найди активный sprint:
   - Посмотри docs/sprints/ — какой sprint открыт (🚧 In Progress)?
   - Открой README.md этого спринта.

3. Определи следующую задачу:
   - Найди в sprint README.md первую задачу со статусом 📋 Planned.
   - Если все задачи в спринте закрыты — предложи открыть следующий sprint.
   - Если sprint не существует — предложи его создать по roadmap.

4. Покажи мне:
   - Текущий sprint и его цель.
   - Следующую задачу (название + краткое описание).
   - Список всех задач спринта с их статусами.

5. Спроси: «Берём в работу [название задачи]?»

После моего «ок»:
- Переключись в режим Plan.
- Создай plan.md по шаблону .methodology/templates/tasks/plan-template.md.
- Перед созданием прочитай соответствующие skills из .methodology/rules/40-skills-router.mdc.
- Покажи plan.md и жди моего подтверждения перед реализацией.
```

---

## Вариант: задать конкретную задачу напрямую

Если ты уже знаешь, что делать, используй такой промпт:

```
Работаем по методологии AI-Driven Development (.methodology/).

Нужно сделать: [опиши задачу одним предложением].
Sprint: [sprint-NN-название или «нужно определить»].

Переключись в режим Plan. Прочитай нужные skills из .methodology/rules/40-skills-router.mdc.
Создай plan.md по шаблону .methodology/templates/tasks/plan-template.md.
Покажи план и жди моего «ок».
```

---

## Что делать, если что-то пошло не так

### Завис на реализации

```
Мы остановились на задаче [название]. Plan.md находится в [путь].
Продолжи реализацию с того места, где остановились.
Покажи текущий статус чекбоксов и что осталось сделать.
```

### Нужно зафиксировать завершённую задачу

```
Задача [название] реализована. DoD пройден. Подтверждаю.
Создай summary.md по шаблону .methodology/templates/tasks/summary-template.md.
Обнови sprint README.md — отметь задачу как ✅ Done.
Если спринт закрыт — обнови roadmap.md.
```

### Нужно открыть новый sprint

```
Sprint [NN] закрыт. Открываем sprint [NN+1] по roadmap.md.
Прочитай docs/roadmap.md, определи задачи следующего sprint'а.
Создай docs/sprints/sprint-[NN+1]-<name>/README.md по шаблону .methodology/templates/sprints/sprint-template.md.
Покажи plan спринта и жди моего «ок».
```
