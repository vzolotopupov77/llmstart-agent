# API Contracts

> **Базовый URL:** `http://localhost:[port]` (локально)
> **Версия API:** `v1` — все эндпоинты бизнес-логики по `/api/v1/...`

---

## Общие конвенции

### Версионирование

Все эндпоинты бизнес-логики: `/api/v1/...`
Служебные эндпоинты (`/health`): без версии.

### Формат запросов и ответов

- `Content-Type: application/json`
- Все тела — JSON, кодировка UTF-8.

### Формат ошибок

```json
{
  "detail": "Human-readable описание ошибки"
}
```

Для ошибок валидации (422):

```json
{
  "detail": [
    {
      "loc": ["body", "field_name"],
      "msg": "field required",
      "type": "missing"
    }
  ]
}
```

### HTTP-коды

| Код | Смысл |
|-----|-------|
| 200 | Успешный ответ (GET, операция без создания ресурса) |
| 201 | Ресурс создан (POST, создающий новую запись) |
| 400 | Ошибка запроса (бизнес-валидация) |
| 404 | Ресурс не найден |
| 422 | Ошибка валидации схемы |
| 503 | Внешний сервис недоступен |

### Пагинация

```json
{
  "items": [...],
  "total": 100,
  "limit": 20,
  "offset": 0
}
```

Пустая коллекция — `200 OK` с `items: []`, не `404`.

## Сводная таблица эндпоинтов

| Метод | Путь | Успешный код | Сценарий |
|-------|------|:---:|---------|
| `POST` | `/api/v1/[resource]` | 201 | [Создать ресурс] |
| `GET` | `/api/v1/[resource]` | 200 | [Получить список] |
| `GET` | `/api/v1/[resource]/{id}` | 200 | [Получить по ID] |

## Эндпоинты

### POST /api/v1/[resource]

**Сценарий:** [Что делает этот эндпоинт.]

**Запрос:**

```json
{
  "field1": "value",
  "field2": 123
}
```

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|---------|
| `field1` | string | ✅ | [описание] |
| `field2` | integer | ❌ | [описание] |

**Ответ `201 Created`:**

```json
{
  "id": "abc-123",
  "field1": "value",
  "created_at": "2026-01-01T00:00:00Z"
}
```

**Ошибки:**

| Код | Условие | Пример `detail` |
|-----|---------|-----------------|
| 422 | Отсутствуют обязательные поля | `"field required"` |
| 400 | [Бизнес-правило нарушено] | `"[сообщение]"` |

---

### GET /api/v1/[resource]

**Сценарий:** [Что возвращает.]

**Query-параметры:**

| Параметр | Тип | Обязательное | Описание |
|----------|-----|:---:|---------|
| `limit` | integer | ❌ | По умолчанию 20, макс 100 |
| `offset` | integer | ❌ | По умолчанию 0 |

**Ответ `200 OK`:**

```json
{
  "items": [
    {
      "id": "abc-123",
      "field1": "value",
      "created_at": "2026-01-01T00:00:00Z"
    }
  ],
  "total": 42,
  "limit": 20,
  "offset": 0
}
```

---

### GET /health

**Ответ `200 OK`:**

```json
{
  "status": "ok",
  "version": "0.1.0"
}
```

| Поле | Тип | Описание |
|------|-----|---------|
| `status` | string | Всегда `ok` если сервис работает |
| `version` | string | Версия из конфига/package |
