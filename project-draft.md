# Сквозной проект курса: AI-агент для LLMStart.ru

## Суть проекта

Разработка **публичного агент-ассистента llmstart.ru** — первой линии продаж и консультаций для потенциальных клиентов. Агент отвечает на вопросы, помогает выбрать продукт, ведёт к оплате и собирает лиды. Реализуется как учебный, но прикладной стенд: архитектура и паттерны — production-grade, интеграции на MVP — моки.

**Принцип:** «один мозг — много каналов» — единое Agent Core, тонкие адаптеры под каналы.

---

## Продуктовая модель

Агент продаёт и консультирует по **B2C-каталогу LLMStart** (5 позиций):

| Продукт | Суть |
|---------|------|
| **ai-agents-combo** | Комбо из 4 программ |
| **vibe-coding-intensive** | Интенсив по Cursor |
| **fullstack-aidd** | AI-driven Fullstack |
| **agents** | Базовый курс по ИИ-агентам |
| **deep-agents** | Продвинутый курс Deep Agents |
| **consultation** | Консультация с экспертом |

Параллельно база знаний и ответы **разделены на B2B и B2C**: корпоративное обучение / разработка под заказ vs курсы для физлиц. Агент учитывает сегмент при поиске в RAG и в формулировках ответа.

---

## Воронка агента (MVP)

1. **Понять запрос** — уточнить потребность, определить B2B / B2C
2. **Подобрать продукт** — каталог, RAG по материалам llmstart.ru
3. **Предложить покупку** — сгенерировать **мок-ссылку** на оплату
4. **Подтвердить оплату** — принять текстовое «оплатил» (**мок**)
5. **Зафиксировать лид** — сохранить контакт в `data/leads.txt`
6. *(далее)* выдача доступа, эскалация на эксперта — в roadmap

## Клиентские интерфейсы

Два канала — один агент, разное представление:

### Веб-виджет (сайт)
- Стиль под llmstart.ru, **SSE-стриминг** ответа
- «Инженерная» визуализация работы агента: **reasoning**, вызванные **инструменты**, статусы шагов — чтобы виджет показывал экспертизу, а не только финальный текст
- На сайте — развилка: **продолжить в виджете** или **перейти в Telegram**

### Telegram
- Long polling, тот же Core через API
- **Форматирование под канал** (HTML вместо сырого Markdown)

API знает `channel` (`web` / `telegram`) — адаптеры тонкие, вся логика в Core.

## Агент Core

| Слой | Решение |
|------|---------|
| Ядро | Python, FastAPI, LangChain ReAct |
| RAG | Файлы `data/b2b/`, `data/b2c/` (PDF, MD), фильтр по аудитории |
| Инструменты | `search_knowledge_base`, `list_b2c_products`, `save_lead`, `create_payment_link`, `confirm_payment` |
| Память | In-memory история диалога |
| Observability | Langfuse (локально в Docker) |
| Каналы | Telegram-бот + Next.js виджет с SSE |
| UI виджета | Стриминг, отображение tools & reasoning |

## Ограничения MVP

- Платежи и CRM — **моки**, не production
- История и диалоги — **in-memory** (сброс при рестарте Core)
- Postgres, эскалация эксперту, мультимодальные чеки, guardrails — следующие итерации
- Фокус курса — **архитектура агента**, tools, RAG, каналы, observability
