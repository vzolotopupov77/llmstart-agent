# Проектирование метрик

Обезличенный конспект. Задачи открытые → строковое `==` не годится (перефразировал ≠ ошибся). Нужны
специальные метрики.

## Главное

**Метрику не выбирают отдельно от датасета.** Она вытекает из двух вещей:
- **что проверяем** (выяснили на анализе — какие способности важны);
- **какой ground truth заложен в запись** (что лежит в поле «ожидаемый результат»).

Поэтому метрику проектируют **вместе** с датасетом, до генерации. Что нашёл на анализе — то и измеряй;
какой эталон положишь — то и сможешь измерить.

> **Граница.** Здесь метрики **проектируем**. Прогон, доведение и тюнинг — это стадия Evaluation.

## Форма эталона определяет метрику

| Что в эталоне записи | Чем меряем |
|---|---|
| категория (из таксономии анализа) | accuracy по категории (можно со ступенями 0/0.5/1.0) |
| вердикт (есть/нет/не знаю) | Verdict Accuracy (бинарно) |
| эталонный текст ответа | LLM-судья (Answer Correctness) |
| «здесь надо отказать» | Rejection Accuracy (без LLM: точный + fuzzy матч фразы-отказа) |
| id нужных фрагментов | retrieval-метрики (Hit Rate@k, MRR, Recall@k) |
| какие инструменты/аргументы + цель | агентные метрики (стадия Evaluation) |

Читать связку можно с двух сторон: **от анализа к метрике** (задача ведёт) и **от формы эталона к
метрике** (данные ведут).

## Метрика из анализа

Классы из анализа становятся и полями эталона, и метриками. Пример — шкала **0 / 0.5 / 1.0** для
«уточняющего» класса:

```python
if not задал_уточняющий_вопрос:
    score = 0.0          # не распознал нехватку контекста
elif категория not in acceptable_categories:
    score = 0.5          # уточнил, но «не про то»
else:
    score = 1.0          # уточнил, и категория верная
```

Ценность — в средней ступени **0.5**: «не переспросил» и «переспросил не про то» — **разные** провалы,
бинарная метрика их бы смешала. Считает **LLM-судья вслепую** (видит только вопрос+ответ, эталон не
видит) → вердикт-категория; сравнение со списком `acceptable_categories` — **в коде**. Если список пуст —
метрика честно вырождается в бинарную.

Полезное наблюдение: **не каждое извлечённое поле становится метрикой**. Часть остаётся аналитикой в
`metadata`; метрика — то, что **сознательно решили измерять**.

## Метрика из формы эталона

- **Есть текст ответа → LLM-судья** (Answer Correctness; построчно нельзя). Судья — сама модель со своими
  ограничениями; на одних judge-метриках выводов не делают (детали — стадия Evaluation).
- **«Надо отказать» → Rejection Accuracy** (без LLM: точный + fuzzy матч фразы-отказа). Самый показательный
  стресс на галлюцинации — на «близких к домену, но не покрытых» вопросах.
- **Есть id нужных фрагментов → retrieval-метрики:**
  - **Hit Rate@k** — попал ли хоть один нужный в первые `k` (1/0);
  - **MRR** — как высоко первый нужный (1/ранг);
  - **Recall@k** — доля нужных в первых `k`.
  - **Precision@k часто выключают**: при 1–2 нужных и `k=5` потолок 0.2–0.4 — метрика не различает хорошее
    и плохое. Осмысленна при 3+ нужных на вопрос.

### Стабильные id чанков — ключевой момент

Где база приходит готовой с **стабильными id** чанков — кладём id в эталон, считаем retrieval дёшево и
объективно. Где базу чанкуешь сам и **не раз навсегда** (меняешь окно/размер/перехлёст) — id нестабилен,
привязывать эталон бессмысленно. Вывод: **где можешь зафиксировать стабильные id — фиксируй**.

## Reference-free: оценка траектории

Когда удобного ground truth нет (особенно у **многоходового агента-собеседника**, где правильных ответов
много, а важна не реплика, а **как агент ведёт весь диалог**):
- эталона-ответа нет; запись хранит **сценарий** + в `metadata` **рубрику** критериев;
- агент проходит сценарий, **LLM-судья получает всю траекторию** и оценивает по каждому критерию;
- судье **дают базу знаний**, иначе оценка вырождается в «звучит складно».

Честный размен: судья без точечного эталона **менее воспроизводим**. Калибровка и пороги — стадия
Evaluation.

## Итог

Лучшая метрика — не самая модная, а та, что честно вытекает из твоих данных. Не бойся **не считать** там,
где данные не позволяют, и **выключать неинформативное**.
